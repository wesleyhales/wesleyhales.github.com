    Open Source

    I started writing code on an Apple IIe when I was 9. We used to get magazines that had coding examples in the back written in Basic and I would sit for hours copying line-by-line, each GOTO and IF/THEN conditional. That was my first open source experience. Over the last decade I've worked with a few languages. Mostly Java, Ruby, and other related server-side frameworks, along with scripting in perl, php, and anything else required to accomplish the task at hand... I've seen a lot of shit come and go. As a UI developer, I consider JavaScript, CSS, and HTML as my strengths.

    User Interface

    I try to adhere to a few principals when dealing with both User Interface design and development. For design it's simple - Make the user's interaction as simple and efficient as possible. All while making it look cool of course. Good User Interface design puts emphasis on goals and completing tasks without drawing too much attention to itself.

    Here we have a fully implemented Firefox theme circa 2009. Over 1.2 million installs and 300k active daily users. During the Firefox 3.6 era, this theme was hot and I received quite a few generous donations from random, kind-hearted people. A very worth while open source effort for me.

    This next theme just ended up being a creative experiment. I never made it to publishing the actual theme on Mozilla's website. After Firefox 4 came along, things began moving so freakin' fast in the browser scene and I couldn't keep up with porting my themes to the new versions.

    Cormada is a project I thought of back in 2010. Bushido came along shortly after I stalled on the idea. The basic idea was: upload any POC or example project you have written and let others run or use it. If it's good, turn it into an OSS project. No idea if it could make money :)
